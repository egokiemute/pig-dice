1. Project Overview

What is Pondice?

Pondice is an interactive, browser-based two-player dice game developed as a final project for the course Web Technologies at Varna Free University, under the supervision of Prof. Boyka Gradinarova and Assist. Madlen Stoycheva.

The name Pondice is a personal combination of the developer's surname Ponda and the word dice — forming a unique brand identity that ties the creator to the project.

"More than just a course requirement, Pondice was inspired by my family. My grandmother once mentioned that she wanted a fun way to improve her confidence with numbers while becoming more comfortable with technology. Wanting to create something meaningful, I designed this game so she and my niece could enjoy playing together on an iPad." — Henry Ponda, Developer

Game Concept

Pondice is a digital adaptation of the classic Pig Dice game. Two players take turns rolling a single die, accumulating points each turn. A player can choose to Hold at any time to bank their current turn score — but rolling a 1 wipes out all unbanked points and passes the turn to the opponent. The first player to reach the chosen target score (25, 50, or 100 points) wins.

The game rewards both risk-taking and strategic thinking, making it accessible for all ages while remaining engaging for experienced players.

Project Scope

The project grew beyond a single game page into a complete multi-page web application consisting of nine pages, a shared component system, persistent data storage, sound effects, background music, a leaderboard, statistics tracking, and full mobile responsiveness.

  • /Home — Landing page with hero, feature highlights, and live statistics strip
  • /game/Play — The game board: avatar selection, game setup modal, dice, score tracking
  • /rules/How To Play — Rules, example turn walkthrough, target score options
  • /project/The Project — Background, inspiration, course context, name origin
  • /about/The Developer — Henry Ponda's biography, affiliations, and education
  • /contact/Contact — Validated feedback form with animated success state
  • /statistics/Statistics — Dashboard with CSS bar chart, win table, and game history
  • /leaderboard/Leaderboard — Trophy podium and full all-time rankings table
  • /settings/Settings — Target score, sound, music, theme — all saved to localStorage

2. Choice of Technology

The project was built exclusively with the three foundational web technologies — HTML5, CSS3, and JavaScript — without any frontend frameworks. This was a deliberate decision to demonstrate a thorough command of the core standards, rather than relying on abstractions.

🌐 HTML5 🎨 CSS3 ⚡ JavaScript (ES6+) 💾 localStorage API 🔊 Web Audio API 🎉 canvas-confetti (CDN) 🔡 Google Fonts

HTML5

HTML5 provides the structural foundation across all nine pages. Each page lives in its own subfolder (e.g., /game/index.html) for clean, readable URLs. The markup uses semantic elements throughout — <nav>, <main>, <section>, <article>, <footer> — which improves accessibility and search engine readability. ARIA attributes (aria-label, aria-current, aria-haspopup) are applied to all interactive navigation elements. HTML5 native form validation (required, type="email", minlength) protects the contact form without JavaScript. Full SEO metadata — Open Graph tags, Twitter Card tags, canonical URLs, and JSON-LD structured data — is included on every page.

CSS3

CSS Custom Properties (variables) power the entire dual-theme system. All colours are defined in a single :root block and overridden in a [data-theme='dark'] block — switching themes requires only toggling one HTML attribute. Flexbox and CSS Grid handle all layout, including the three-column game board, the responsive navigation, the podium display, and the statistics bar chart. CSS @keyframes animate the dice shake, modal entrance, roll log fade-in, and the homepage dice spin. Glassmorphism — combining rgba transparency with backdrop-filter: blur() — creates the frosted-glass card effect. CSS clamp() enables fluid typography that scales proportionally between mobile and desktop. The mobile hamburger menu is implemented in pure CSS using a hidden checkbox and sibling combinator selectors — no JavaScript required. Navigation dropdowns use :hover and :focus-within for keyboard accessibility.

JavaScript

All game logic, UI interaction, and data management is written in vanilla JavaScript (ES6+). The Web Audio API generates all sound effects (dice roll ticks, loss buzz, win chime) using OscillatorNode and GainNode — no audio files are required. A dedicated music manager module handles two background tracks that cross-fade over 0.8 seconds using HTMLAudioElement, with a synthesised ambient fallback generated entirely in code when files are absent. The localStorage API persists four data categories: game statistics, leaderboard rankings, game settings, and theme preference — all surviving page refreshes and browser restarts. Animated dice rolling uses setInterval to cycle through dice faces before landing on the final result, with a CSS shake animation triggered at the moment of reveal. The confetti celebration on a win uses the canvas-confetti library loaded via CDN.

3. Key Functionality

Game Mechanics

Players set their name (up to 12 characters), select an emoji avatar from eight options, and choose a target score before each game. On their turn, a player clicks Roll Dice — the die animates through several random faces before landing on the final result (1–6). Scores accumulate until the player clicks Hold to bank them, or rolls a 1 and loses the turn's unbanked points. The first player to reach or exceed the target wins, triggering a confetti animation, a three-note victory chime, and an automatic leaderboard update. Keyboard shortcuts (R / H / N) allow full play without a mouse.

Data Persistence

All data is stored in the browser's localStorage, meaning statistics and rankings build up over multiple sessions without any server or database. The Statistics page displays a pure CSS animated bar chart of wins per player. The Leaderboard renders a trophy podium for the top three players, followed by a complete win-rate table for all players ever recorded.

Responsive Design

The site is fully responsive across mobile, tablet, and desktop. Below 600px, the three-column game board stacks vertically, the navigation collapses into a hamburger menu, and all font sizes and button heights adapt automatically. The hamburger animation — bars rotating into an × — is achieved purely with CSS transitions and no JavaScript.

Accessibility & SEO

ARIA labels, aria-current indicators, and keyboard-navigable dropdowns are applied throughout. Every page includes a unique meta description, Open Graph and Twitter Card tags for social sharing, and a JSON-LD structured data block on the homepage identifying the site as a free web game application for search engines.

Audio System

Sound effects (roll, lose turn, win, click) are synthesised using the Web Audio API — no audio files required. Background music cross-fades between a calm ambient track during the setup phase and a more energetic track during active gameplay. Both the sound effects and background music have independent toggles in Settings, and a 🎵 mute button is available directly in the game navigation.

Dark Mode

A full dark theme is available on every page. The colour scheme shifts from the teal-to-olive earth-tone gradient to a deep forest-green gradient, with card backgrounds changing from near-white glass to near-black glass. The preference is saved to localStorage and restored on every page load.
